# Система сбора отзывов через QR-коды + Telegram-бот

## Архитектура системы (полный обзор)

### Принцип работы

```
┌─────────────────────────────────────────────────────────────┐
│  МАГАЗИН (плакат с QR-кодом)                                │

                         │ скан QR
                         ↓
┌─────────────────────────────────────────────────────────────┐

│  Пользователь выбирает:                                     │
│  ┌──────────────────┐    ┌──────────────────┐               │
│  │ Заполнить форму  │    │ Перейти в TG-бот │               │
│  └────────┬─────────┘    └────────┬─────────┘               │
└───────────┼───────────────────────┼─────────────────────────┘
            │                       │
            │                       │ deep-link с shop_id
            │                       ↓
            │           ┌─────────────────────────┐
            │           │   Telegram-бот          │
            │           │   /start shop_42        │
            │           │   FSM-диалог            │

│  - Сохранение в БД                                          │
└────────────────────────┬────────────────────────────────────┘
                         ↓
        ┌────────────────┴─────────────────┐
        ↓                                  ↓
┌───────────────────┐              ┌──────────────────────┐
│  Supabase (БД)    │              │  Telegram (админы)   │
```

## Технологический стек (рекомендую)

| Компонент | Технология | Почему |
|-----------|-----------|--------|
| **Веб-форма** | React + Vite + TypeScript + Tailwind + shadcn/ui + Framer Motion | Современный стек, легко собрать в один HTML и положить на Битрикс через файловую систему |
| **Бот** | Python 3.11 + aiogram 3.x | Лучшая FSM для диалогов, асинхронность, твой стек |
| **Backend API** | FastAPI | Один процесс с ботом, валидация Pydantic, async |
| **БД** | Supabase (PostgreSQL) | Бесплатно, веб-интерфейс, аналитика, бэкапы |
| **QR-коды** | Python `segno` | Бесплатно, генерация SVG с логотипом |
| **Хостинг бота** | Твой VPS (Linux) | 0₽, полный контроль, без лимитов времени |
| **Хостинг формы** | Битрикс (статика) | По твоему запросу, ZIP в файловую систему |
| **HTTPS** | Caddy / Nginx + Let's Encrypt | Автоматический SSL |
| **Защита от спама** | Honeypot + Cloudflare Turnstile (бесплатно) | Без надоедливых капч |

**Стоимость: 0₽/месяц** (всё на бесплатных тарифах + твой существующий VPS).

## Схема данных (Supabase / PostgreSQL)

```sql
-- Магазины (заполняется один раз при создании плакатов)
CREATE TABLE shops (
    id              SERIAL PRIMARY KEY,
    code            VARCHAR(20) UNIQUE NOT NULL,    -- "minsk-01", зашит в QR
    name            VARCHAR(200) NOT NULL,           -- "Продтовары на Победителей, 5"
    address         TEXT NOT NULL,
    city            VARCHAR(100),
    manager_chat_id BIGINT,                          -- TG ID директора (для адресной маршрутизации)
    is_active       BOOLEAN DEFAULT true,
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_shops_code ON shops(code) WHERE is_active = true;

-- Отзывы
CREATE TABLE feedbacks (
    id              SERIAL PRIMARY KEY,
    shop_id         INTEGER REFERENCES shops(id),
    
    -- Данные клиента
    name            VARCHAR(100),
    contact_method  VARCHAR(20) CHECK (contact_method IN ('telegram','phone','email','none')),
    contact_value   VARCHAR(200),  -- @username, +375..., email
    
    -- Содержание отзыва
    category        VARCHAR(30) CHECK (category IN ('praise','complaint','suggestion','question')),
    rating          SMALLINT CHECK (rating BETWEEN 1 AND 5),
    message         TEXT NOT NULL,
    photo_urls      TEXT[],   -- массив путей в Supabase Storage
    
    -- Согласие на обработку ПД
    consent_given   BOOLEAN NOT NULL DEFAULT false,
    consent_at      TIMESTAMPTZ,
    consent_ip      INET,
    
    -- Источник
    source          VARCHAR(20) CHECK (source IN ('web_form','telegram_bot')),
    user_agent      TEXT,
    
    -- Статус обработки
    status          VARCHAR(20) DEFAULT 'new' 
                    CHECK (status IN ('new','in_progress','answered','closed')),
    assigned_admin  BIGINT,                 -- TG ID админа, который взял в работу
    
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    answered_at     TIMESTAMPTZ
);

CREATE INDEX idx_feedbacks_status ON feedbacks(status, created_at DESC);
CREATE INDEX idx_feedbacks_shop ON feedbacks(shop_id, created_at DESC);

-- Ответы админов (история переписки)
CREATE TABLE feedback_replies (
    id              SERIAL PRIMARY KEY,
    feedback_id     INTEGER REFERENCES feedbacks(id) ON DELETE CASCADE,
    admin_chat_id   BIGINT NOT NULL,
    admin_name      VARCHAR(100),
    message         TEXT NOT NULL,
    delivered       BOOLEAN DEFAULT false,
    delivery_error  TEXT,
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

-- Rate limiting (защита от спама на уровне БД)
CREATE TABLE rate_limits (
    ip_hash         VARCHAR(64) PRIMARY KEY,
    request_count   INTEGER DEFAULT 1,
    window_start    TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_rate_limits_window ON rate_limits(window_start);

-- Админы бота
CREATE TABLE admins (
    chat_id         BIGINT PRIMARY KEY,
    name            VARCHAR(100),
    role            VARCHAR(20) DEFAULT 'admin' CHECK (role IN ('admin','manager','viewer')),
    notify_shops    INTEGER[],   -- какие магазины уведомляют этого админа (NULL = все)
    is_active       BOOLEAN DEFAULT true,
    added_at        TIMESTAMPTZ DEFAULT NOW()
);

-- Аудит (для требований законодательства о ПД)
CREATE TABLE audit_log (
    id              SERIAL PRIMARY KEY,
    feedback_id     INTEGER REFERENCES feedbacks(id),
    action          VARCHAR(50),     -- 'created','viewed','answered','deleted'
    actor_chat_id   BIGINT,
    details         JSONB,
    created_at      TIMESTAMPTZ DEFAULT NOW()
);
```

## Структура полей формы (минимум по закону + UX)

### Обязательные поля
1. **Текст отзыва** (мин. 10 символов, макс. 2000)
2. **Категория**: благодарность / жалоба / предложение / вопрос
3. **Оценка**: звёзды 1-5 (опционально, но сильно повышает аналитику)
4. **Согласие на обработку персональных данных** (чекбокс с ссылкой на политику)

### Опциональные (показываются только если человек хочет ответ)
- **Имя** (как обращаться)
- **Способ связи**: Telegram / телефон / email / не нужен ответ
- **Контакт** (валидируется по выбранному способу)

### Скрытые
- `shop_code` — из URL `?shop=minsk-01`
- `honeypot` — невидимое поле, бот его заполнит → отбросим
- `submission_time` — если форма заполнена быстрее 3 секунд → бот
- `consent_ip` + `consent_at` — для юридической защиты

## Юридические требования (РБ — закон о персональных данных)

Так как ОАО «Продтовары» работает в РБ, нужно соблюсти **Закон РБ № 99-З «О защите персональных данных»**:

1. **Согласие должно быть явным** — отдельный чекбокс, не предзаполненный
2. **Политика обработки ПД** — отдельная страница, ссылка из чекбокса
3. **Цель обработки** — указана в политике («рассмотрение обращений клиентов»)
4. **Срок хранения** — рекомендую 3 года (срок исковой давности), потом автоудаление
5. **Право на удаление** — кнопка «Удалить мои данные» в боте
6. **Уведомление в НЦЗПД** — компания должна быть в реестре операторов ПД (это уже должно быть у юристов)

Я добавлю в код шаблон политики обработки ПД — юрист отредактирует под Продтовары.

## Защита от спама и атак (без капчи на форме)

### 4 уровня защиты:

**1. Honeypot (на форме)**
```html
<input name="website" style="position:absolute;left:-9999px" tabindex="-1" autocomplete="off">
```
Боты заполняют все поля → если поле непустое, отбрасываем.

**2. Time-trap**
Если форма отправлена быстрее 3 секунд после загрузки — бот.

**3. Cloudflare Turnstile** (только при подозрительной активности)
Невидимая капча, активируется при множественных запросах с одного IP.

**4. Rate limit на backend**
- Не более 3 отзывов с одного IP за 10 минут
- Не более 10 отзывов с одного магазина за минуту (защита от DDoS на конкретный магазин)

## Логика бота: режимы работы

### Режим 1: Клиент пишет боту
```
/start shop_42  →  Бот: «Здравствуйте! Вы оставляете отзыв о магазине 
                  "Продтовары на Победителей, 5". Всё верно?»
                  [Да] [Нет, другой магазин]
                  ↓
                  «Что вас привело к нам?»
                  [👍 Благодарность] [⚠️ Жалоба] [💡 Предложение] [❓ Вопрос]
                  ↓
                  «Оцените магазин от 1 до 5 ⭐»
                  ↓
                  «Опишите подробнее»
                  ↓
                  «Хотите получить ответ? Куда написать?»
                  [Сюда, в Telegram] [Не нужен ответ]
                  ↓
                  «Согласны на обработку данных? [политика]»
                  [✅ Согласен] [❌ Не согласен — отменить]
                  ↓
                  ✅ Спасибо! Ваш отзыв №123 принят.
```

### Режим 2: Админ отвечает на отзыв
```
В админ-чат приходит:
┌─────────────────────────────────────┐
│ 🆕 Отзыв #123                       │
│ 📍 Победителей, 5                   │
│ ⚠️ Жалоба ⭐⭐                       │
│ 👤 Анна (TG: @anna)                 │
│                                     │
│ «Очередь на кассе 20 минут...»      │
│                                     │
│ [✏️ Ответить] [✅ Закрыть] [👤 Взять]│
└─────────────────────────────────────┘
                  ↓
Админ нажимает «Ответить» → бот спрашивает текст ответа
                  ↓
Бот пересылает ответ клиенту в TG (или на email)
                  ↓
Статус отзыва → 'answered'
```

### Команды админа:
- `/stats` — статистика за день/неделю/месяц
- `/pending` — список неотвеченных отзывов
- `/shop minsk-01` — все отзывы по магазину
- `/export` — выгрузка в Excel
- `/help` — список команд

## Безопасность и приватность (чек-лист)

- ✅ HTTPS только (Caddy + Let's Encrypt автоматически)
- ✅ Все секреты в `.env`, не в коде
- ✅ Хеширование IP (SHA-256) — храним только хеш для rate limit
- ✅ Pydantic-валидация всех входов (защита от инъекций)
- ✅ CSP-заголовки на форме (защита от XSS)
- ✅ Rate limit на endpoint `/api/feedback`
- ✅ Минимум данных от клиента (только то, что нужно)
- ✅ Автоудаление через 3 года (cron-job в боте)

- ✅ Логи только в JSON, без ПД
- ✅ Honeypot + time-trap от ботов
- ✅ Telegram bot token хранится только на сервере
- ✅ Webhook secret для Telegram (защита от подделки запросов)
- ✅ Аудит-лог всех действий с отзывами
